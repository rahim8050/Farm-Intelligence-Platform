# Security Policy

## Supported versions

Security fixes are applied to the current main development line and to any
public release branches that are explicitly maintained.

## Reporting a vulnerability

Do not file public issues for security vulnerabilities.

Instead, report security concerns privately to the maintainers using the
project's preferred private contact path, or open a private security report if
the hosting platform supports it.

Include:

- A short description of the issue
- The affected endpoint, module, or document
- Steps to reproduce
- Any proof of concept or log output, if safe to share

## What not to send

- Passwords
- JWTs
- API keys
- HMAC secrets
- Database dumps
- Any other secret material

## Response expectations

The maintainers will triage the report, confirm impact, and coordinate a fix or
mitigation before public disclosure.
