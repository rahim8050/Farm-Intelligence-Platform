  Phase 3 Step 2: Retry-After Header Parsing

  Added support for server-suggested retry delays on 429 responses:

  - New helper: `parse_retry_after(response_headers)`
    - Supports numeric delay (e.g., `Retry-After: 120` → 120 seconds)
    - Supports HTTP-date format (e.g., `Retry-After: Wed, 21 Oct 2026
      07:28:00 GMT` → calculated delay)
    - Case-insensitive header lookup
    - Returns `0.0` for past dates, `None` for absent/invalid

  - Updated `should_retry(exception, response_headers=None)`:
    - Extracts `Retry-After` delay for 429 responses only
    - Non-429 responses ignore the header
    - Backward compatible: existing callers work without changes
    - Returns `RetryDecision` with server-suggested delay when available

  Phase 3 Step 3: Admin Endpoint

  Added endpoint to manually reset circuit breakers:

  - Circuit breaker registry in `ndvi/circuit_breaker.py`:
    - `register_circuit_breaker(cb)` — registers instance by engine name
    - `get_circuit_breaker(engine)` — lookup by engine name
    - `list_circuit_breakers()` — returns copy of full registry

  - All engines now register on init:
    - `StacClient` → `"stac"`
    - `SentinelHubEngine` → `"sentinelhub"`
    - `SentinelHubRasterEngine` → `"sentinelhub_raster"`

  - New endpoint: `POST /api/v1/ndvi/circuit-breaker/reset/`
    - Auth: `IsAdminUser` only
    - Request: `{"engine": "stac"|"sentinelhub"|"sentinelhub_raster"}`
    - Response: envelope with `previous_state` and `new_state`
    - 400: invalid engine name
    - 403: non-admin user
    - 404: engine not initialized
    - OpenAPI fully documented with request/response schemas
