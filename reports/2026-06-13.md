## Done

### Bullet 1 — Critical bug: swapped `index_type` in refresh views

- `ndvi/views.py`: **`NdviRefreshView`** (NDVI refresh) was passing `index_type="NDWI"` — removed so it defaults to `"NDVI"`.
- `ndvi/views.py`: **`NdwiRefreshView`** (NDWI refresh) was omitting `index_type` entirely — added `index_type="NDWI"`.
- This meant NDVI refreshes triggered NDWI V2 quality, and NDWI refreshes ran NDVI V2 quality. Both inverted.

### Bullet 2 — `?representation=v2` for NDWI API views

- Added `index_type` param (default `"NDVI"`) to `_inject_v2_observations()` and `_inject_single_v2_observation()` with `index_type` filter on `NdviDerivedObservation` queries.
- Wired representation reading + V2 injection in `NdwiTimeseriesView.get()` and `NdwiLatestView.get()` (both cached and fresh paths).
- Also fixed missing `index_type="NDWI"` filter in the NDWI latest view observation query.

### Bullet 3 — NDWI-specific outlier and null thresholds

- `v2_quality.py`: Added optional threshold override kwargs to `_check_outlier()` (`outlier_threshold`, `accept_threshold`, `vpf_threshold`) and `_check_null_conditions()` (`vpf_reject_threshold`, `low_confidence_threshold`, `min_rolling_context`).
- `quality_ndwi.py`: `build_ndwi_v2_observation()` now passes NDWI-specific values: outlier deviation=0.25, accept=0.70, VPF=0.60, VPF reject=0.25, low confidence=0.45, min context=4.

### Bullet 4 — Remove `source .venv/bin/activate` from README

- README: Replaced the two-line setup block (with `source .venv/bin/activate`) with a single `uv sync` command and a note that all subsequent commands use `uv run`.
- `scripts/daily-report.sh`: Changed `.venv/bin/python` to `uv run python`.
