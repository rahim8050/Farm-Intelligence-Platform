  Root Cause Analysis: STAC API Failures

  Problem: All STAC API requests were failing with:
    "STAC API returned invalid JSON (status=200, body=<html>Request Rejected...)"

  Diagnosis:
    - Copernicus STAC API's WAF (F5 BIG-IP) was blocking all requests from our IP.
    - Support ID: 6518495712231534107 (and similar IDs in repeated attempts).
    - WAF returned HTML error page instead of JSON, causing JSON parse failures.
    - Confirmed IP ban by testing multiple User-Agent strings and methods — all blocked.
    - Even base URL health checks (/v1/) were rejected.

  Root cause: IP address blacklisted by Copernicus WAF due to excessive request rate.

  Solution Part 1: OpenJPEG Decode Error Fix

  Problem: rasterio failed with `opj_get_decoded_tile failed` when reading
  remote JPEG2000/COG assets directly via HTTP.

  Fix:
    - Added `_local_asset_context()` context manager to `ndvi/stac_client.py`.
    - Downloads remote assets to temp files before processing.
    - Local file paths are returned unchanged (no download needed).
    - Temp files are cleaned up automatically via TemporaryDirectory.
    - Eliminates OpenJPEG streaming decode errors completely.

  Code changes:
    - `_is_remote_href()`: Detects HTTP(S) asset URLs.
    - `_local_asset_context()`: Downloads remote assets to temp files.

  Solution Part 2: WAF Detection & Early Warning

  Added early detection of WAF block responses:

  - New exception: `StacWafBlockedError` (inherits from `StacUpstreamError`).
    - Non-retryable (prevents wasted retry attempts).
    - Extracts and logs WAF support ID for diagnostics.

  - Helper functions:
    - `_looks_like_waf_response()`: Detects common WAF indicators:
      - "Request Rejected"
      - "Web Application Firewall"
      - "Your support ID"
      - "The requested URL was rejected"
    - `_extract_waf_support_id()`: Parses support ID from WAF response via regex.

  - Integration: `_parse_json_response()` checks Content-Type and body for
    HTML/WAF patterns before attempting JSON parse.

  ———

  Solution Part 3: Circuit Breaker Pattern

  Added `_CircuitBreaker` class to prevent wasteful retries when upstream is
  unreachable:

  States:
    - CLOSED: Normal operation (requests pass through).
    - OPEN: Circuit tripped (all requests fail immediately).
    - HALF_OPEN: Testing recovery (one request allowed through).

  Behavior:
    - Opens after `NDVI_STAC_CIRCUIT_BREAKER_THRESHOLD` (default: 3) failures.
    - Auto-recovers after `NDVI_STAC_CIRCUIT_BREAKER_TIMEOUT_SECS` (default: 300s).
    - Transitions HALF_OPEN→CLOSED on success, HALF_OPEN→OPEN on failure.
    - Logs state transitions for observability.

  Settings (added to `.env` and `.env.example`):
    - `NDVI_STAC_CIRCUIT_BREAKER_THRESHOLD=3`
    - `NDVI_STAC_CIRCUIT_BREAKER_TIMEOUT_SECS=300.0`

  ———

  Solution Part 4: Proxy Support

  Added optional proxy configuration to bypass IP bans:

  - New setting: `NDVI_STAC_PROXY_URL` (default: unset).

  - Example: `NDVI_STAC_PROXY_URL=http://proxy.example.com:8080`
  - Useful for:
    - Temporary IP ban bypass.
    - Corporate network requirements.
    - Geographic restrictions.
