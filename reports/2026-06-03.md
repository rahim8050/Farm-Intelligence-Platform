### 2. CI fix: migration index-name hashes

CI's `makemigrations --check --dry-run` step was about to auto-generate a
`0004_rename_*` migration to fix two index names in `0003` that I had hand-named
with guessed hash suffixes. The hashes in Django's auto-generated names are
derived from the field list; my guesses were `4b0e1c` / `b0d5f2` but Django
wanted `973d05` / `bb44e7`. Fixed in place in
`radio/migrations/0003_station_health_check.py:35, 78` so the file still has
the same migration number and no rename migration is needed. Verified with:

```
$ python manage.py makemigrations --check --dry-run
No changes detected
```
