Report: Farm State Endpoint + NDVI Coverage Metrics (as of Mar 22, 2026)

  Scope of work

  - Added farm state endpoint to expose NDVI-derived farm classification.
  - Fixed metrics endpoint to include farm state coverage data.
  - Added comprehensive test coverage for farm state logic.

  ———

  Farm State Classification

  The system now classifies farms into states based on NDVI observations:

  - establishment: Sparse but healthy vegetation (mean NDVI 0.25-0.40).
  - full_canopy: Dense canopy with strong vigor (mean NDVI > 0.40).
  - decline: NDVI trend declining over 30-60 day window.
  - growth: NDVI trend increasing over 30-60 day window.
  - unknown: Insufficient data to classify.

  State metadata includes description and recommended action for each state.

  ———

  Coverage Calculation

  - Computes percentage of farm area covered by valid NDVI data.
  - Uses STAC raster sampling with configurable parameters:
    - Asset bands: B04_10m (RED), B08_10m (NIR).
    - Sample size: Configurable (default from STAC settings).
    - Date window: Configurable (default 3 days around target date).
  - Caches coverage results with TTL to avoid repeated STAC calls.
  - Implements distributed lock to prevent concurrent coverage calculations.

  Configuration:
    - DEFAULT_COVERAGE_TTL_SECONDS: 21600 (6 hours).
    - DEFAULT_COVERAGE_LOCK_SECONDS: 300 (5 minutes).
    - DEFAULT_ESTABLISHMENT_MEAN_THRESHOLD: 0.25.
    - DEFAULT_FULL_CANOPY_MEAN_THRESHOLD: 0.40.
    - DEFAULT_COVERAGE_THRESHOLD: 0.30 (30% coverage required).

  Endpoint

  GET /api/v1/farms/{farm_id}/ndvi/state/

  Response includes:
    - state: Farm classification (establishment/full_canopy/decline/growth/unknown).
    - state_description: Human-readable description.
    - state_action: Recommended action.
    - coverage: Percentage of farm with valid NDVI data (0.0-1.0).
    - mean_ndvi: Average NDVI value.
    - trend_ndvi: NDVI trend over window (positive = growth, negative = decline).
    - sample_count: Number of valid NDVI samples.
    - target_date: Date used for classification.
    - engine: NDVI engine used (stac/sentinelhub).
