Report: STAC Raster Fallback + Asset Resolution Improvements (as of Apr 3, 2026)

  Scope of work

  - Improved STAC raster fallback handling for Copernicus STAC items.
  - Changed default asset names from B04/B08 to B04_10m/B08_10m for Sentinel-2 L2A.
  - Added candidate item ranking with cloud cover and date proximity.
  - Enhanced error reporting with structured reason codes for raster failures.

  ———

  Problem

  - STAC raster_png jobs were failing with "Raster not found" on Copernicus.
  - Copernicus STAC items use suffixed asset names (B04_10m, B08_10m) instead of plain B04/B08.
  - Asset resolution logic didn't handle fallback to suffixed variants.
  - Error messages lacked context about why raster was not found.

  ———

  Solution

  Asset naming:
    - Changed DEFAULT_ASSET_RED from "B04" to "B04_10m".
    - Changed DEFAULT_ASSET_NIR from "B08" to "B08_10m".
    - Aligned with Copernicus Sentinel-2 L2A asset naming convention.

  Candidate ranking:
    - Added _rank_candidate_items() function.
    - Ranks by: (1) lowest cloud cover, (2) closest date, (3) datetime.
    - Filters out items outside date window before ranking.
    - Returns ordered list of candidates for processing.

  Error reporting:
    - Added structured reason codes to ValidationError:
      - "no_items": STAC search returned no results.
      - "no_best_item": No items within date window.
      - "missing_assets": Items exist but lack required bands.
    - Included context in error response:
      - reason: Failure category.
      - detail: Human-readable message.
      - code: Machine-readable error code.

  Fallback handling:
    - try/except around load_ndvi_array() for each candidate.
    - Skips items with processing failures, tries next candidate.
    - Raises StacUpstreamError with summary of all failures if all candidates fail.
    - Distinguishes between missing_assets vs empty_stats failures.

  ———

  Files Modified

  - config/settings.py: Changed NDVI_RASTER_ENGINE_NAME default to use NDVI_ENGINE.
  - ndvi/raster/stac_compute_engine.py: Major refactor (242 lines added, 50 removed).
    - _rank_candidate_items(): New ranking function.
    - Updated render_png() to use ranking + fallback logic.
    - Enhanced _raise_raster_not_found() with structured errors.
  - ndvi/tests/test_ndvi_raster_engines.py: Added 126 lines of test coverage.
    - test_stac_compute_engine_no_items_raises_not_found.
    - test_stac_compute_engine_no_best_item_raises_not_found.
    - test_stac_compute_engine_missing_assets_raises_not_found.
    - test_stac_compute_engine_missing_assets_logs_context.
    - test_stac_compute_engine_missing_stats_raises_not_found.
    - test_stac_compute_engine_skips_processing_failure_and_uses_next_item.
    - test_stac_compute_engine_all_processing_failures_are_retryable.

  ———

  Processing Flow

  1. Search STAC for items in bbox + date window.
  2. If no items: raise "no_items" error.
  3. Rank candidates by cloud cover + date proximity.
  4. If no candidates in window: raise "no_best_item" error.
  5. For each candidate:
     a. Resolve RED and NIR asset hrefs.
     b. If assets missing: log warning, try next candidate.
     c. Load NDVI array from raster assets.
     d. If processing fails: log error, try next candidate.
     e. If NDVI stats empty: log warning, try next candidate.
     f. If successful: encode PNG and return.
  6. If all candidates fail: raise StacUpstreamError with failure summary.

  ———

  Assessment

  - Raster PNG generation now resilient to asset naming variations.
  - Copernicus STAC items work correctly with B04_10m/B08_10m assets.
  - Candidate ranking ensures best imagery selected (lowest cloud, closest date).
  - Structured errors help diagnose why raster generation failed.
  - Fallback logic prevents single bad item from blocking entire request.
