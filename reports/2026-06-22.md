# Daily Report — 2026-06-22

## Summary

Added MODIS NDWI support via MOD09GA surface reflectance (Green band 4 + NIR band 2), resolving the previous MODIS gap for NDWI. Then rewrote the NDMI architecture doc from an index-specific feature addition into a full production-grade spectral analytics platform architecture covering 13 design areas (formula registry, generic compute engine, async compute, COG storage, provenance, multi-level caching, provider circuit breakers, SLOs, tenant isolation, event-driven evolution, science/platform separation).

## Changes

### MODIS NDWI support
- `ndvi/engines/modis.py`: Removed `UnsupportedIndexError` for NDWI. When `index_type="NDWI"`, uses MOD09GA collection (`modis-09ga-061`) with `sur_refl_b04` (Green, 500m) + `sur_refl_b02` (NIR, 500m) + `state_1km` QA. Existing NDVI via MOD13Q1 unchanged.
- `config/settings.py`: Added `NDVI_MODIS_NDWI_STAC_COLLECTION`, `NDVI_MODIS_GREEN_BAND`, `NDVI_MODIS_NIR_BAND`, `NDVI_MODIS_NDWI_QA_BAND` settings.
- `ndvi/tests/test_ndvi_edge_cases.py`: Updated `test_get_engine_ndwi_modis` to expect success instead of `UnsupportedIndexError`.
- Docs: Updated `00-overview.md`, `01-architecture.md`, `06-risks.md` — MODIS is no longer "skipped" for NDWI.

### NDMI architecture rewrite
- `docs/agritech/ndmi/01-architecture.md`: Rewritten from 289 lines (index-specific) to 1075 lines (production-grade spectral platform). Covers: Formula Registry, Band Registry, `SpectralComputeEngine`, async compute with Celery queues, COG storage, provenance, multi-level caching, provider circuit breakers, SLOs, tenant isolation, event-driven evolution, science/platform separation, migration strategy, risk register, production readiness checklist.

### NDWI index_type dedup fix
- `ndvi/services.py`: Added `index_type` to `hash_request()` hash and signature; added `index_type` to `enqueue_job()` dedup filter so NDWI jobs don't collide with NDVI jobs.
- `ndvi/views.py`: Added `index_type` param to `_lookup_raster_not_found_reason()`, wired `"NDVI"`/`"NDWI"` at all call sites.
- `ndvi/streams.py`: Added `index_type` to `build_stream_payload()` so NDWI identity survives Redis stream serialization.
- `ndvi/management/commands/consume_ndvi_stream.py`: `_route_ndvi_job()` now checks payload `index_type` and dispatches to `run_ndwi_job` for NDWI jobs.
- `ndvi/tests/test_ndvi_streams.py`, `ndvi/tests/test_ndvi.py`, `ndvi/tests/test_ndvi_raster_png.py`: Updated test fixtures and assertions for `index_type`.
