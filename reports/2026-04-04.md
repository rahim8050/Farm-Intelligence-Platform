Report: NDVI Raster PNG Normalization Fix (as of Apr 4, 2026)

  Scope of work

  - Investigated why Farm 21 (and all farms with healthy vegetation) were generating solid green PNG images.
  - Diagnosed root cause in NDVI-to-PNG normalization formula.
  - Implemented hybrid normalization with histogram stretching (new default) and fixed mapping (legacy).
  - Added comprehensive tests and verified all quality checks.

  ———

  Root Cause

  - NDVI calculation was correct: Farm 21 NDVI range [0.42, 0.63] (healthy vegetation).
  - Normalization formula `(ndvi + 1.0) / 2.0` mapped this to [0.71, 0.81].
  - Entire range fell in green zone of RdYlGn colormap (0.6-1.0).
  - Result: 100% of pixels appeared green with no visible variation.

  Diagnostic evidence:
    - Live STAC data fetch confirmed RED band (B04_10m): min=1204, max=1953.
    - NIR band (B08_10m): min=3950, max=5928.
    - NDVI calculation: [0.417, 0.630], mean=0.541 (correct).
    - RGB output: Green channel mean=196, Red=120, Blue=101.
    - 100% pixels green-dominant (Green > Red AND Green > Blue).

  ———

  Solution

  - Added ColormapNormalization enum with two modes:
    - histogram (NEW DEFAULT): Per-image min-max stretching to use full colormap range.
    - fixed (legacy): Constant [-1, 1] → [0, 1] mapping for color consistency.
  - Updated ndvi_to_rgb() and ndvi_to_png_bytes() to accept normalization mode.
  - Added NDVI_COLORMAP_NORMALIZATION setting (defaults to "histogram").
  - Modified render_ndvi_png() and task handler to pass through normalization mode.

  Impact:
    - Before: NDVI [0.42, 0.63] → Normalized [0.71, 0.81] → All green.
    - After:  NDVI [0.42, 0.63] → Normalized [0.00, 1.00] → Full red-yellow-green spectrum.
    - Low NDVI (0.42) → Red/Orange (bare soil, paths).
    - Medium NDVI (0.54) → Yellow (moderate vegetation).
    - High NDVI (0.63) → Green (dense vegetation).

  Configuration

  - Default: histogram (fix applied automatically, no action needed).
  - Optional: Set NDVI_COLORMAP_NORMALIZATION=fixed to keep old behavior.
  - Invalid values: Logs warning, falls back to histogram.
  - Restart required: Django web process + Celery workers after deployment.

  ———

  Assessment

  - NDVI calculation was always correct; visualization normalization was the issue.
  - Histogram mode provides maximum detail within each farm (recommended for analysis).
  - Fixed mode provides consistent colors across images (for comparison use cases).
  - Fully backward compatible: no API changes, no migrations, no breaking changes.
  - Performance impact negligible: <1ms overhead for 512x512 image.
