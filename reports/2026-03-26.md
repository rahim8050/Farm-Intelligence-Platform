Report: Nextcloud Farms schema 404 + Redis cache (as of Mar 26, 2026)

  Scope of work

  - Investigated Nextcloud admin UI error: "Unable to parse farm schema response."
  - Verified behavior after moving Nextcloud cache from APCu to Redis.

  ———

  Findings

  - Schema URL injected into the admin page is correct:
      - /nextcloud/index.php/apps/weather_apis/api/v1/admin/farms/schema
  - Redis is healthy and reachable (redis-cli + PHP ping as www-data).
  - Route exists and matches in Nextcloud router:
      - weather_apis.adminfarms.getschema
  - UI still failed because the schema response was not JSON, consistent with stale cache/route artifacts.

  ———

  Resolution

  - Cleared Redis cache keys; schema request started returning JSON.
  - Farms UI schema loading unblocked.
  - Kept Redis as the cache backend to preserve worker token caching.

  ———

  Notes / follow-ups

  - If this reappears: confirm route with `occ router:match`, then clear Redis keys and hard-refresh the UI.
  - Avoid APCu-only local cache for Nextcloud workers unless `apc.enable_cli=1` is set.
