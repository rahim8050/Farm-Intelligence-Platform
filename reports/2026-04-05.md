  Server Resource Tuning (NEW)

  Created comprehensive deployment guide: `monitoring/tuning-configs/DEPLOYMENT_GUIDE.md`
  with 8 phases of tuning for the entire stack:

  - **PHP-FPM** (`monitoring/tuning-configs/nextcloud-php-fpm.conf`):
    - Max 15 workers (prevents OOM from unlimited spawning).
    - Restart after 500 requests (prevents memory leaks).
    - 512MB per-request memory limit.
    - OPcache tuned for 256MB shared memory.

  - **MySQL/MariaDB** (`monitoring/tuning-configs/mysql-tuning.cnf`):
    - InnoDB buffer pool capped at 1GB.
    - Max connections reduced to 100 (from 151 default).
    - Query cache enabled (64MB, MariaDB only).
    - Slow query logging enabled (2s threshold).

  - **Redis** (`monitoring/tuning-configs/redis-tuning.conf`):
    - Max memory 512MB with allkeys-lru eviction.
    - RDB snapshots disabled (cache-only mode).
    - TCP backlog increased to 2048.

  - **Celery workers** (`config/celery.py`):
    - worker_max_tasks_per_child=100 (auto-restart to prevent leaks).
    - worker_max_memory_per_child=512000 (512MB hard cap).
    - task_time_limit=300 (5 min hard kill).
    - task_soft_time_limit=240 (4 min soft kill).

  - **Prometheus/Grafana/Loki** (`docker-compose.monitoring.yml`):
    - Prometheus retention: 7 days / 2GB (prevents disk fill).
    - Loki retention: 7 days / 1GB.
    - Redis memory limits: 512MB max.
    - Added Loki container to monitoring stack.

  - **Prometheus alerts** (`monitoring/prometheus/alerts.yml`):
    - MemoryUsageHigh (80%) / MemoryUsageCritical (95%).
    - DiskSpaceLowRoot (85% usage).
    - RedisMemoryNearLimit / RedisEvictionStarted.
    - NextcloudLogFileLarge (>50MB).

  STAC Request Throttling

  Added request rate limiting to `StacClient` to prevent WAF blocks:

  - `NDVI_STAC_REQUEST_INTERVAL_SECS` (default: 10s between requests).
  - `NDVI_STAC_JITTER_MIN_SECS` / `NDVI_STAC_JITTER_MAX_SECS` (random jitter).
  - Prevents pattern detection by Copernicus WAF.
  - Applied to every STAC API call with configurable values.
