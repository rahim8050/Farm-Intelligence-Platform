### Phase 7 — Fallback stations, description, metadata URL, analytics, now-playing (`17c5f204`)

- **Models** (`radio/models.py`):
  - `Station.description` (TextField) and `Station.metadata_url` (URLField) — rich station metadata.
  - `StationAnalytics(station, date, listeners, peak_listeners, duration_listened, listen_seconds, ...)` with `UniqueConstraint(station, date)` and composite index `(station, -date)`.
  - `NowPlaying(station, track_artist, track_title, stream_title, started_at, duration_seconds)` — `OneToOneField(Station)`, replaces itself on each poll.
  - `ListeningHistory` already existed (Phase 3).
- **Services** (`radio/services.py`):
  - `DEFAULT_FALLBACK_STATION_MAP` — when a station is unavailable, the system falls back to a known-good alternative (configurable via `RADIO_FALLBACK_STATION_MAP`).
  - `get_fallback_station_map` / `get_fallback_station` — resolves rules; `None` means "use defaults", `{}` disables all fallbacks.
  - `_parse_icy_stream_title` / `fetch_icy_metadata` — best-effort ICY metadata parsing (16 KiB cap, failures logged at info, never raised).
  - `refresh_now_playing` / `get_now_playing` — polls ICY stream headers, updates `NowPlaying` row.
  - `rollup_station_analytics` / `get_station_analytics` — daily aggregation into `StationAnalytics` (beat at 00:15 and 12:00 with `lookback_days=2`).
- **Tasks** (`radio/tasks.py`): `rollup_station_analytics_task` and `refresh_now_playing_task` with `shared_task` + autoretry.
- **Settings** (`config/settings.py`): `RADIO_NOW_PLAYING_REFRESH_INTERVAL_SECONDS=300` (env-overridable), `RADIO_FALLBACK_STATION_MAP: dict[str, str] | None = None`, 3 new Celery beat entries.
- **Views** (`radio/views.py`): `StationNowPlayingView` and `StationAnalyticsView` — `GET /api/v1/radio/<id>/now-playing/` and `/analytics/` with envelope schemas.
